# Global AI Atlas Design

**Topic:** `全球 AI 模型版图` 单页展示界面  
**Style Direction:** `Atlas Prime`  
**Date:** `2026-05-16`

## Goal

构建一个高端科技品牌风格的单页展示站，主题为“全球 AI 模型版图”。页面需要同时满足两件事：

1. 视觉上有强烈的全球化、数据化、前沿科技感。
2. 内容上能让用户快速识别主要 AI 模型阵营及其区域特征。

页面不是百科式信息看板，而是“品牌首页 + 行业概览”的混合形态。重点先建立气质，再承载精炼内容。

## Audience

- 希望快速了解全球主流 AI 模型生态的技术用户
- 需要截图、整理、写博客的开发者或技术博主
- 对大模型行业格局感兴趣的普通访客

## Design Principles

### 1. 品牌感优先

页面首先要像一个成熟的科技品牌官网，而不是普通信息罗列页面。版式、留白、色彩、层次、动态都要服务于“高级感”。

### 2. 识别度优先

页面需要同时展示：

- 风格化主徽标：负责统一视觉语言
- 官方品牌字标/图标感元素：负责让用户一眼认出 `OpenAI / Anthropic / Google / Meta / Mistral / DeepSeek`

### 3. 信息适量

页面只保留少量高价值模块，避免堆砌卡片和说明文字。每个区块都必须有明确职责。

### 4. 全球版图叙事

整页视觉核心围绕“全球模型势力分布”展开，使用地球、轨道、数据网格、区域阵营等元素，而不是抽象科技背景。

## Page Architecture

页面采用单页纵向结构，分为 4 个主要区域。

### Section 1: Hero 首屏

职责：

- 建立第一眼视觉冲击
- 传达“全球 AI 模型版图”主题
- 展示风格化主徽标和品牌识别带

内容：

- 左侧：主标题、副标题、简短说明文案
- 中下：品牌识别条，放置主要模型阵营名称
- 右侧：风格化地球徽标与轨道光效
- 背景：深海蓝渐变 + 数据网格 + 柔和发光层

预期效果：

- 一打开页面就能看出“这是一个关于全球 AI 模型格局的高端展示页”

### Section 2: Global Overview 全球总述

职责：

- 用最短内容概括全球模型格局
- 将复杂行业信息压缩成可快速理解的三段式表达

内容：

- 北美前沿：强调闭源旗舰、多模态、推理能力
- 欧洲创新：强调开源协作、轻量化、研究气质
- 亚洲跃迁：强调效率优化、工程落地、快速迭代

表现形式：

- 3 张高质感内容卡片
- 每张卡片使用区域标签、短句描述、轻量视觉强调

### Section 3: Model League 代表阵营

职责：

- 展示代表性模型或组织
- 让页面不仅“好看”，也具备明确识别信息

内容建议：

- `OpenAI`
- `Anthropic`
- `Google`
- `Meta`
- `Mistral`
- `DeepSeek`

表现形式：

- 统一风格卡片或标识块
- 每个项目配一句非常短的定位语
- 不做长篇介绍，不做技术参数堆叠

### Section 4: Trend Signal 趋势观察

职责：

- 给页面增加“观点”
- 让内容更像技术博客/行业观察的入口页

内容：

- 一句总结式观察标题
- 1 到 3 条简短趋势语句

示例方向：

- 推理能力正在成为新一轮竞争核心
- 开源模型正在重塑区域创新节奏
- 全球模型竞争正在从参数规模转向系统能力

## Visual Language

### Color System

- 主背景：深海蓝、夜空蓝、冷黑蓝
- 强调色：电光青、科技蓝、微量暖金
- 文本：高亮白 + 次级蓝灰

目的：

- 深色背景承载全球数据氛围
- 青蓝色系建立未来科技感
- 暖金只做极少量点缀，避免页面发冷、发空

### Typography

- 标题：强对比、高辨识度、偏科技/编辑感
- 正文：清晰克制，强调阅读舒适性
- 标签：小号大写或高字距样式，增强品牌陈列感

### Shape & Surface

- 大圆角容器
- 半透明玻璃层
- 细描边和微发光边缘
- 不采用普通矩形白卡片

### Motion

只做轻量、高级、连续性的动效：

- 地球徽标缓慢呼吸光
- 数据网格轻微扫光
- 卡片入场上浮
- 鼠标悬停时出现边缘提亮

禁止：

- 高频闪烁
- 过度旋转
- 大幅位移
- 会破坏高级感的夸张动画

## Logo Strategy

### 风格化主徽标

主徽标不直接复刻任何现有公司标志，而是设计一个“全球模型中枢”感的抽象地球标志：

- 球体
- 经线/纬线
- 轨道环
- 核心高光

作用：

- 提供统一的品牌感
- 避免页面像“拼贴资讯站”

### 官方识别元素

页面中对主要品牌使用可识别的字标/名称呈现方式，保证用户知道页面在讲哪些模型阵营。

原则：

- 不追求复杂图像复刻
- 更偏字标化、品牌陈列化
- 与整体页面视觉保持统一

## Content Tone

文案采用“短句式品牌表达”，不是百科说明。

文案特征：

- 句子短
- 节奏强
- 少术语堆砌
- 一眼可截图

示例语气：

- 北美定义前沿推理高地
- 欧洲推动开源创新密度
- 亚洲正在重写效率曲线

## Data Flow / Interaction

页面以展示为主，交互以轻量体验增强为主：

- 首屏按钮或锚点滚动
- 卡片 hover 反馈
- 徽标轻量动画
- 可选的小范围视差感

不包含：

- 复杂筛选器
- 动态请求接口
- 后端数据源

## Files To Modify

- `index.html`
  - 承载页面结构与主要内容区块
- `main.js`
  - 承载轻量动效、滚动或 hover 增强逻辑

当前阶段不新增复杂依赖，保持纯静态页面实现。

## Testing Expectations

需要验证：

1. 页面在桌面端能够完整显示 Hero、内容卡片和趋势区块
2. 页面在移动端不会出现严重溢出或布局断裂
3. 动效不会影响可读性
4. `index.html` 与 `main.js` 可以直接本地打开或由简单静态服务器访问

## Out of Scope

本轮不做以下内容：

- 接入真实模型 API
- 复杂交互数据图表
- 多页面站点
- 后台管理功能
- 长篇技术数据库式模型列表

## Success Criteria

如果实现完成后满足以下条件，则设计目标达成：

1. 页面第一眼具备明显“高端科技官网”气质
2. 用户能快速识别页面主题是“全球 AI 模型版图”
3. 页面能看出主要模型阵营的全球分布叙事
4. 页面包含风格化主徽标和官方识别元素的结合
5. 内容简洁但不空洞，适合作为技术博客配图或首页展示
